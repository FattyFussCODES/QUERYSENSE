# QuerySense

QuerySense is a **schema-aware Natural Language → SQL (Text-to-SQL)** system that allows users to interact with relational databases using natural language instead of writing SQL queries manually.

The system interprets user queries, maps them to database schema elements, generates SQL statements automatically, executes them in the database, and returns structured results.

> Database engine: **SQLite (default demo)**, **MySQL (optional)**

## Overview

Relational databases require users to write SQL queries to retrieve and manipulate data. However, many users such as analysts, managers, and domain experts may not be familiar with SQL syntax.

QuerySense provides a natural language interface that converts English queries into SQL statements and executes them in the database automatically.

### Example

**User Query**

Show students with marks above 80

**Generated SQL**

```sql
SELECT name
FROM students
WHERE marks > 80;
```

The query is executed in the database and results are returned to the user.

## Problem Statement

Traditional relational database systems require knowledge of SQL to retrieve information. This creates several challenges:

- Non-technical users cannot easily access data
- Organizations depend on developers for simple queries
- SQL syntax errors are common
- Query complexity increases with large database schemas

These limitations reduce the accessibility of data stored in relational databases.

## Solution

QuerySense introduces a schema-aware natural language interface that enables users to retrieve database information using plain English queries.

The system performs the following steps:

1. Accepts a natural language query
2. Processes the query using NLP techniques
3. Maps keywords to database schema elements
4. Generates an SQL query
5. Executes the query in the database
6. Returns structured results

This approach reduces the need for manual SQL writing.

## Key Features

- **Natural Language Querying**: Users can query databases using plain English (example: “Show students enrolled in AI course”).
- **Automatic SQL Generation**: Converts natural language queries into SQL statements.
- **Schema Awareness**: Dynamically reads database schema metadata to identify tables, columns, and relationships.
- **Join Detection**: Automatically generates SQL joins when multiple tables are referenced.
- **Query Logging**: Stores user query, generated SQL, execution time, and timestamp.
- **SQL Preview**: Users can see the SQL query generated by the system.

## Installation

1. Clone the repository:

```bash
git clone https://github.com/yourusername/querysense.git
```

2. Navigate to the project directory:

```bash
cd querysense
```

3. Create a virtual environment:

```bash
python -m venv venv
```

4. Activate the environment:

**Windows:**

```powershell
venv\Scripts\activate
```

**Linux / macOS:**

```bash
source venv/bin/activate
```

5. Install dependencies:

```bash
pip install -r requirements.txt
```

## Quick Demo (SQLite)

Initialize the demo database and run a query:

```powershell
python main.py --init-db
python main.py "show students with marks above 80"
python main.py "count students"
python main.py "top 2 students"
```

Run the API:

```powershell
uvicorn api.app:create_app --factory --reload
```

Then POST to `http://127.0.0.1:8000/query` with JSON:

```json
{ "query": "students in AI" }
```

## Running the Application

Start the system:

```powershell
python .\main.py
```

When prompted, enter a natural language query.

Example:

```text
Enter your query: Show students with marks above 80
```

If database initialization is enabled in your flow, running `main.py` will:

- create the database if it doesn’t exist
- apply [database/schema.sql](database/schema.sql)
- apply [database/sample_data.sql](database/sample_data.sql) (if present)
